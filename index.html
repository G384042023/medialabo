「Webコンテンツ作成」の成果

本テーマについて

HTML・CSS・JavaScriptの役割を理解し、メディアを活用したWebページの設計・制作力を養うとともに、 情報工学の基礎を応用して実験に取り組み、論理的思考力や課題解決力、チームでの協働力を高め、 成果を適切に伝える力を身につける。

プロジェクト課題について

今回のプロジェクト課題では、世界各地の都市を選択すると、その地域の最新の天気情報をページ上に表示する Webアプリケーションを作成しました。HTMLで基本構造を設計し、CSSで背景画像やレイアウトを整え、 JavaScriptでは天気データの取得・表示を動的に実装しました。API通信によって都市の天気を取得し、 都市名や緯度・経度、天気の概要、気温や湿度、風速などの詳細情報をページに出力する仕組みです。 特に苦労した点は、取得したJSONデータを適切に解析して、必要な項目のみを抽出し、DOM要素として正しく表示させる処理です。 また、再検索時に前の情報が重複しないよう、要素の削除処理も工夫しました。実装が完成した際には、 プログラムの流れをしっかり理解できたと感じ、実用的なWeb開発の基礎力が身についたと思います。

プロジェクト課題で作成したページ: project.html

課題3-1では、1から10の乱数を「答え」として設定し、ユーザーが予想した数と比較して正解かどうか を判定するゲームの基本部分を作成しました。予想は固定値で、hantei 関数を4回実行し、結果は すべてコンソールに出力されます。if文による分岐を用いて、正解・不正解のメッセージを適切に 表示する処理を学びました。

課題4-1では、数当てゲームにテキストボックスと回答ボタンを設置し、ユーザーの入力に応じて正解か どうかを判定する処理をJavaScriptで実装しました。ボタンがクリックされるたびに、入力された値を 取得して乱数と比較し、判定結果や予想回数はページ上に動的に表示されます。正解した場合には成功メ ッセージが、3回不正解だった場合には答えが表示され、4回以上になるとゲーム終了のメッセージが表示 される仕組みとなっており、ゲームとしての完成度を高めました。

各授業回の学習事項

第1回 WWW と HTML

今回の学習では、HTMLの基本的な書き方について理解を深めた。特に、段落を示す際には <p>と</p>で囲むことが大切であり、これはHTML構造の基本となる。

他にも、空白の挿入方法（&nbsp;など）、特殊文字の表現方法、コメントの記述方法 （<!-- コメント -->）など、多様な要素が存在する。

さらに、ハイパーリンクを作成する際には、href属性でリンク先を指定し、 title属性で補足情報を加え、target属性でリンクの開き方を設定できる。

HTMLの基礎を学ぶ中で、特に重要だったのは<p>タグを使って文章の区切りを明確にする点だ。 HTMLには様々な要素があり、例えば改行や空白を表現する方法、特殊文字（記号など）を表示する方法、 コメントを記述してコードの見通しを良くする手段などがある。

また、リンクを設定する際にはhrefでURLを指定し、titleでリンク内容の補足説明を加え、 target属性を使えば新しいタブで開くようにもできる。このようにHTMLはシンプルながらも多機能であることを学んだ。

演習の成果: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回の授業では、HTML ファイルと CSS を別々に管理し、CSS を使ってスタイルを一括で変更できるという点が強調されました。これにより、HTML に直接手を加えずに外観を調整できます。

たとえば、見出しの色（h1など）を変更したい場合、CSS ファイルにルールを追加するだけで済み、作業の効率が大きく向上します。

もう一つの重要な内容はレイアウト技術です。Flexbox は要素を横並びにする際に便利ですが、細かな幅の調整はやや手間がかかる面もあります。

一方で、Grid レイアウトはページを行と列に分けて領域を作り、その中に要素を配置する仕組みです。これにより、中心に要素を置くようなデザインも簡単に実現できます。

こうした CSS の技術を活用することで、ページ内の構造を視覚的に整理し、背景色などを用いて区切ることが可能になります。

演習の成果: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

JavaScript に関する基礎は第3回で紹介され、スクリプトの記述方法やHTMLとの連携について理解を深めました。外部ファイルとしての活用が推奨され、コードの整理がしやすくなります。

Firefox の開発者ツールを使えば、プログラムの実行結果を即座に確認できます。エラーがあった場合も、対応するコードの位置が特定できるため、修正がスムーズです。

先生は例を交えながら、JavaScript の実行方法や、簡単な構文の使い方をデモ形式で見せてくださり、実際に手を動かしながら学習する流れとなっていました。

演習の成果: オブジェクトの参照

第4回 DOM と JavaScript

第4回では、Webページにおけるユーザー操作への反応、すなわち「イベント処理」の基礎を学びました。input要素のtype属性による使い分けや、フォーム部品としての テキストボックスやボタンの配置方法について実践的に理解を深めました。

また、JavaScriptを活用してHTML要素を取得し、テキストや属性を操作する方法を学び、 ユーザーの入力に応じた動的な処理が可能になることを体験しました。ボタンが押された際に特定の処理を 実行するためのイベントリスナーの記述方法や、DOMを用いた要素の取得・変更といった、 インタラクティブなページ作成の第一歩を踏み出しました。

演習の成果: 演習4-10のHTMLファイル

第5回 イベント処理

第5回では、JavaScriptを用いてHTMLの要素を動的に操作するスキルを実践的に身につけました。 createElement() により新たな要素を作成し、insertAdjacentElement() を活用して任意の場所に挿入する方法を学習。 また、setAttribute() を使って属性の設定や変更を行い、remove() で不要な要素の削除も行いました。 さらに、ulタグと複数のliタグを組み合わせて、複数要素を一括で生成し配置する方法も取り扱いました。 最終的には、これらの手順を関数changeDom() にまとめ、ボタン操作一つでページの内容が動的に更新される仕組みを作り上げました。

演習の成果: 演習5-7 DOMプログラミング②

まとめ:

今回の「Webコンテンツ作成」の講義を通じて、私はHTML・CSS・JavaScriptの基礎から応用までを体系的に学び、 自分のアイデアを形にするWebページ制作の技術を身につけることができました。授業の後半では、学んだ知識を活用して、 実際に動作するWebアプリケーションを開発するプロジェクト課題に取り組みました。

プロジェクト課題では、世界の都市を選択すると、対応する天気情報がページ上に表示されるWebアプリケーションを作成しました。 HTMLではページの構造を整え、CSSでは背景画像の設定やレイアウトの調整を行い、JavaScriptではAPI通信とDOM操作 による動的な情報表示を実装しました。APIで取得したJSON形式の天気データから、都市名・緯度経度・天気概要・気温・ 湿度・風速などの情報を抽出して、画面上に分かりやすく出力する仕組みを構築しました。

工夫した点は、操作のしやすさと情報の見やすさを両立させることです。検索方法を自由入力ではなく選択式にすることで、 ユーザーの入力ミスを防ぎ、直感的に操作できるようにしました。また、データ表示部分はレイアウトや配色を工夫し、 情報が多くても読みやすくなるように意識しました。さらに、再検索のたびに前回の表示内容を削除する処理を加えることで、 ページが煩雑にならないよう配慮しました。

苦労した点としては、JavaScriptにおける非同期通信（axiosによるAPI取得）の理解や、JSONデータの扱い方が挙げ られます。初めて見る形式のデータ構造を理解するのに時間がかかり、console.logを何度も使って値を確認しながら少 しずつ実装を進めました。また、Javaを学んだ経験があるため、JavaScriptの文法との違いに混乱し、変数のスコープ や関数の書き方などでエラーを頻発させてしまいました。さらに、DOM操作による要素の生成・挿入・削除処理では、細 かな構文ミスが大きなバグにつながるため、慎重に確認しながら進める必要がありました。

強調したいのは、プログラムの正確性だけでなく、ユーザー視点での使いやすさを意識した設計ができるようになった 点です。Web制作は見た目や動作がユーザーに直接伝わるため、レイアウトや配色の工夫、データの整った見せ方が非 常に重要だと実感しました。そのため、情報の並び方や余白の使い方、背景画像とのバランスなど細部にも気を配りま した。単に「動く」だけではなく、「見やすく、使いやすい」ページを作ることの難しさと大切さを学びました。

この講義を通して、私はWeb制作の技術だけでなく、エラーへの対処力や論理的思考力、そして何より試行錯誤を繰 り返しながら一つの作品を完成させる達成感を得ることができました。初めはコードが思うように動かず、 何度もエラーに悩まされましたが、先生や友人の助けも借りながら、最後には自分の力で解決できたという自信につながりました 。今後はさらに応用的な技術にも挑戦し、今回得た知識と経験を活かして、より高品質で実用的なWebページを作れるよう 努力していきたいと思います。
